# Tempo

A quiet, local-first practice tracker for hobby musicians.

Tempo is a small web app for tracking practice sessions, repertoire, and the
patterns those add up to over time. Everything lives in your browser's
IndexedDB — there is no account, no server, no cloud sync. Export a JSON
snapshot whenever you want a backup.

## Features

- **Today** — start, pause, and save a session with instrument, piece, intent,
  tempo, and a short end-of-session reflection. Streak front-and-centre.
- **Library** — manage pieces with composer, status (learning / polishing /
  maintenance / shelved), difficulty, key, tempo target, and notes. Per-piece
  detail view shows total minutes and a 60-day practice bar.
- **Instruments** — track multiple instruments, mark one as primary.
- **Sessions** — grouped, filterable history of everything you've logged.
- **Stats** — GitHub-style 365-day heatmap, current/longest streak, intent
  split, instrument split, time-of-day histogram, pieces in rotation.
- **Goals** — weekly minutes, "piece ready by date X", and "play instrument Y
  N times per week", each with live progress bars.
- **Settings** — theme (light / dark / system), JSON export & import, clear
  all data.

## Stack

- Next.js 16 (App Router) · React 19 · TypeScript strict + `noUncheckedIndexedAccess`
- Tailwind CSS 4 with a warm amber/rose palette on a slate base
- `idb` over native IndexedDB for persistence
- Vitest + Testing Library for the test layer

## Getting started

```bash
npm install
npm run dev
```

Then open <http://localhost:3000>.

On first launch the app is empty. Use **Seed with example data** on the home
screen to load 30 days of synthetic practice across two instruments and four
pieces — handy for previewing every screen at once.
